# Dev-Cycle: Cycle

You are executing a single improvement cycle. This can be invoked manually via `/dev-cycle:cycle` or automatically by the `start` skill's loop.

If `.workflow/state.json` does not exist, stop:
> No active dev-cycle session. Run `/dev-cycle:start` first to initialize the project profile.

Read `.workflow/state.json` to get the cycle number and project profile.

---

## Phase 1 — Audit (Teams 1 & 2 in Parallel)

Launch **both** agents simultaneously:

1. **Code Auditor** (agent: `code-auditor`) — scan for bugs, dead code, consolidation, security issues
2. **Improvement Scout** (agent: `improvement-scout`) — scan for UX gaps, dark mode, a11y, performance, features

Both agents receive the project profile for context.

Wait for both to complete, then:

1. **Combine** all findings from both agents into a single list
2. **Deduplicate** — if both agents flagged the same issue, keep the more detailed one
3. **Sort by priority** — P0 first, then P1, P2, P3
4. **Select top 5** — take the 5 highest-priority findings for this cycle
5. If **zero findings** from both agents → report "Codebase is in good shape — no issues found this cycle" and end the cycle

Display the selected findings to the user:
> **Cycle {N} — Selected {X} tasks:**
> 1. [P{x}] {title} ({category})
> 2. ...

## Phase 2 — Task Board Creation

Create `.workflow/tasks.md` with this format:

```markdown
# Dev Cycle Tasks — Cycle {N}
Generated: {ISO timestamp}

## Summary
- Total: {X} | To Do: {X} | In Progress: 0 | Done: 0 | Blocked: 0

### [P{x}] {Task title}
- **Status**: To Do
- **Source**: Team 1 (auditor) | Team 2 (scout)
- **Category**: {bug | security | dead-code | consolidation | ux | dark-mode | api-keys | qol | feature | a11y | performance | docs | code-smell}
- **Branch**: {fix|feat}/cycle-{N}-{slug}
- **PR**: pending
- **Files**: {file1}:{line}, {file2}:{line}
- **Description**: {What to do and why}
- **QA Result**: pending

---
```

**Branch naming**:
- Bugs, security, dead-code, consolidation, code-smell → `fix/cycle-{N}-{slug}`
- Everything else (ux, feature, qol, a11y, performance, docs, dark-mode) → `feat/cycle-{N}-{slug}`

The `{slug}` should be a short kebab-case summary (max 5 words).

## Phase 3 — Development (Team 3, Sequential)

First, make sure you're on the default branch and up to date:
```bash
git checkout {default_branch}
git pull origin {default_branch}
```

For **each task** in the task board (in priority order):

### 3a. Setup
1. Update tasks.md → set task status to **"In Progress"**
2. Create the branch: `git checkout -b {branch_name}`
3. Rebase on latest: `git fetch origin {default_branch} && git rebase origin/{default_branch}`
   - If the branch already exists, append `-v2` (or `-v3`, etc.)

### 3b. Develop
4. Launch the **developer** agent with:
   - Task title, description, priority
   - Affected files list
   - Branch name
   - Project profile (language, framework, test runner, linter commands)

5. After the developer agent completes:
   - Update tasks.md → set status to **"Done"**, record the commit hash
   - Push the branch: `git push -u origin {branch_name}`
   - Create a PR: `gh pr create --title "{task title}" --body "{description} --- Generated by dev-cycle (Cycle {N})" --base {default_branch} --head {branch_name}`
   - Record the PR number in tasks.md

### 3c. QA
6. Launch the **qa-reviewer** agent with:
   - Task title, description
   - Branch name
   - Default branch name
   - Project profile

7. Based on QA verdict:
   - **PASS** → update tasks.md QA Result to "PASS", move to next task
   - **FAIL** → give developer agent **1 retry** with the QA failure details
     - If retry passes QA → "PASS", continue
     - If retry fails again → update status to **"Blocked"**, QA Result to "FAIL — {reason}"
   - **ESCALATE** → update status to **"Blocked"**, QA Result to "ESCALATE — {reason}", alert user

### 3d. Cleanup
8. Return to default branch: `git checkout {default_branch}`
9. Pull latest: `git pull origin {default_branch}`
10. Proceed to next task

## Error Handling

| Scenario | Action |
|----------|--------|
| State file missing | Stop: "Run `/dev-cycle:start` first" |
| Zero findings | End cycle gracefully with "codebase in good shape" message |
| Developer fails to commit | Mark task Blocked, continue to next |
| QA fails after retry | Mark task Blocked, continue to next |
| `gh pr create` fails | Log error, mark PR as "failed", continue |
| Merge conflicts during rebase | Developer resolves; if can't → Blocked |
| Branch already exists | Append `-v2` suffix |
| All tasks blocked | End cycle, report to user |
| Protected file modification needed | Developer marks Blocked with reason |
